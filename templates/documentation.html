项目简介

Hadoop 集群部署系统是一款专业的自动化部署与管理工具，支持全自动、半自动和手动三种部署模式，适用于企业、科研和个人用户。

  • 三种部署模式：全自动、半自动、手动配置，满足不同需求
  • 文件上传支持：自定义上传 Hadoop 和 Java 安装包
  • Web 可视化界面，实时日志与进度显示
  • 多镜像源支持，跨平台（Linux/Windows）
  • 内置安全配置与最佳实践

系统要求

  • 操作系统：Windows 10/11 或 Linux (Ubuntu 18.04+, CentOS 7+)
  • Python 版本：3.8 及以上
  • 内存：至少 4GB（推荐 8GB+）
  • 磁盘空间：10GB 以上
  • 网络：稳定的互联网连接

安装与快速开始

  1. 克隆仓库：git clone https://github.com/violet27-chf/Hadoop.git
  2. 进入目录并创建虚拟环境：cd Hadoop && python3 -m venv venv && source venv/bin/activate
  3. 安装依赖：pip install -r requirements.txt
  4. （可选）运行安装脚本：./setup.sh 或 setup.bat
  5. 创建必要目录：mkdir -p uploads logs
  6. 启动服务：python app.py 或 ./start.sh
  7. 浏览器访问 http://localhost:5000，开始部署

使用指南

  • 选择部署模式：全自动、半自动或手动
  • 配置服务器信息、选择软件版本、上传自定义包（可选）
  • 点击"开始部署"，系统自动执行部署流程
  • 支持实时日志、进度监控和部署结果反馈

配置说明

所有部署参数均可在 config.yaml 文件中自定义，包括集群节点、端口、Hadoop/Java 版本、副本数等。详细字段说明请参考文档注释。

hadoop:
  version: 3.3.6
  nodes:
    - host: 192.168.1.101
      role: namenode
    - host: 192.168.1.102
      role: datanode
java:
  version: 8
hdfs:
  replication: 3

故障排除

  • 虚拟环境激活失败： 检查命令和 Python 版本
  • 依赖安装失败： 升级 pip，或使用国内镜像源
  • 端口被占用： 修改 app.py 中的端口号
  • 文件上传失败： 检查格式、大小和网络
  • 部署失败： 检查服务器连接、权限和日志
